# مسار · أداتك الشخصية للوقت وتطوير الذات

تطبيق ويب فردي مبني بـ Next.js، يتتبع وقتك بعجلة يوم تفاعلية، يدير مهامك، ويحلل نمطك بالذكاء الاصطناعي (Claude). البيانات تُحفظ سحابياً عبر Supabase مع نسخة محلية احتياطية في المتصفح.

> ملاحظة مهمة: التطبيق يعمل فوراً بالتخزين المحلي فقط بدون أي إعداد. خطوات Supabase وClaude أدناه اختيارية لكنها تفعّل المزامنة السحابية والتحليل الذكي.

---

## ١. التشغيل محلياً على جهازك

تحتاج Node.js نسخة ١٨ أو أحدث.

```bash
npm install
npm run dev
```

ثم افتح المتصفح على `http://localhost:3000`.

في هذه المرحلة يعمل التطبيق بالكامل (تتبع، مهام، رسوم بيانية) بالتخزين المحلي. لتفعيل التحليل الذكي والمزامنة السحابية، أكمل الخطوات التالية.

---

## ٢. إعداد قاعدة البيانات (Supabase)

1. أنشئ حساباً مجانياً على [supabase.com](https://supabase.com) ثم أنشئ مشروعاً جديداً.
2. من القائمة الجانبية افتح **SQL Editor** ثم **New query**.
3. انسخ كامل محتوى ملف `supabase/schema.sql` والصقه ثم اضغط **Run**. هذا ينشئ الجداول (categories, entries, tasks, reports, gamify) والفئات الافتراضية.
4. من **Settings > API** انسخ قيمتين: **Project URL** و **anon public key**.

---

## ٣. مفتاح التحليل الذكي (Claude)

1. أنشئ مفتاحاً من [console.anthropic.com](https://console.anthropic.com) تحت **API Keys**.
2. هذا المفتاح سري ويبقى على الخادم فقط، لا يصل للمتصفح أبداً.

---

## ٤. ملف البيئة

انسخ `.env.example` إلى ملف جديد اسمه `.env.local` واملأ القيم:

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
ANTHROPIC_API_KEY=sk-ant-your-key
```

أعد تشغيل `npm run dev`. الآن تظهر علامة السحابة الخضراء في أعلى التطبيق، وتعمل المزامنة والتحليل.

---

## ٥. النشر على Vercel

1. ارفع المشروع إلى مستودع على GitHub.
2. ادخل [vercel.com](https://vercel.com) واربط حسابك بـ GitHub ثم اختر **Import Project** وحدد المستودع.
3. في إعدادات المشروع على Vercel، تحت **Environment Variables**، أضف المتغيرات الثلاثة نفسها من `.env.local`.
4. اضغط **Deploy**. خلال دقائق يصبح لديك رابط مباشر يعمل بكامل وظائفه.

> يعمل النشر على Netlify بنفس الطريقة: اربط المستودع، أضف متغيرات البيئة، وانشر.

## بنية المشروع

```
masar/
  app/
    layout.js            هيكل الصفحة وRTL والخطوط
    page.js              التطبيق الكامل (كل الواجهات)
    globals.css          الهوية البصرية (أسود وذهبي)
    api/analyze/route.js مسار خادمي يستدعي Claude بأمان
  components/
    DayWheel.js          عجلة اليوم التفاعلية
    styles.js            الأنماط المشتركة
  lib/
    supabase.js          عميل Supabase
    store.js             طبقة التخزين (سحابي + محلي)
    helpers.js           أدوات مساعدة وشارات
  supabase/
    schema.sql           مخطط قاعدة البيانات
```

## الوحدات

- **اليوم**: عجلة يوم تفاعلية ٢٤ ساعة، اضغط أي قوس لتفاصيله، مع قسم تطوّرك اليومي.
- **الصلاة**: أوقات الصلوات الخمس لمدينة الكويت (حساب فلكي داخلي بطريقة أم القرى، يعمل بدون إنترنت)، الصلاة القادمة مع عد تنازلي، إشعار عند دخول الوقت، زر "تمت الصلاة" يسجّلها، ومهام دينية يومية (الاستغفار 1000 مرة، قراءة القرآن 30 دقيقة) فيها عدّاد زمني يسجّل الدقائق قبل الإكمال.
- **تركيز**: مؤقت مرن تتحكم فيه يدوياً (تزيد وتنقص بخطوات دقيقة أو خمس دقائق)، مع وضعين: دراسة ونشاط عام. كل جلسة تُحفظ بشكل دائم في ذاكرة التايمر. يتضمن تقريرين منفصلين (تقرير دراسة وتقرير عام)، وتحديات التزام يومية فردية، وتحدي الروبوتات الثلاثة (ناجح، متوسط، متكاسل) الذين يتحركون على مدار اليوم ويتفاعلون مع تقدّمك.
- **المهام**: دفتر ذكي لإضافة وتعديل وحذف وتصنيف المهام، مع زر أساسيات اليوم الذي يضيف مهام انضباط جاهزة.
- **أنجز**: وحدة ذكاء اصطناعي تقترح تحديات ومشاريع ومسارات تعلّم مرتبطة بهواياتك وتخصصك، مع روابط بحث عن فيديوهات تعليمية لكل عنصر.
- **التقارير**: يومية وأسبوعية وشهرية برسوم شريطية ودائرية وخطية، مع تصدير PDF.
- **التحليل**: تحليل ذكي حيّ لأدائك يشمل نسبة الالتزام بالمهام، معدل التركيز، عدد الإنجازات، والتوازن بين الدراسة والعمل والروحانية.
- **التخصيص**: هويتك (من أنا، هواياتي، تخصصي)، الفئات والألوان، والشارات.

> ملاحظة عن الأرقام: كل الأرقام في الموقع تُعرض بالإنجليزية مع الحفاظ على النصوص العربية.
> ملاحظة عن أوقات الصلاة: تُحسب لمدينة الكويت بطريقة أم القرى وقد تختلف دقيقة أو دقيقتين عن تقويم وزارة الأوقاف. تعتمد الصلاة القادمة على ساعة جهازك، فتأكد أن منطقتك الزمنية على توقيت الكويت.

## ملاحظات

- لا يوجد نظام تسجيل دخول. التطبيق مصمم لمستخدم واحد، وكل البيانات تحت معرّف `solo`. عند الحاجة لإضافة مصادقة لاحقاً، تُستبدل القيمة `solo` بـ `auth.uid()` في `store.js` ومخطط قاعدة البيانات وسياسات RLS.
- التخزين المحلي يبقى نسخة احتياطية دائمة حتى لو تعطّل الاتصال بالسحابة.
- كل النصوص العربية خالية من الشرطات الطويلة، وفق هويتك في التصميم.
